# 文档目录索引

这个目录按用途分层，方便后续智能体快速判断应该读哪些文档。

## 优先阅读

1. 根目录 `Agent.md`、`memory.md`、`Context.md`：当前协作规则、稳定决策和最近接力上下文。
2. `docs/README.md`：判断报告、设计、计划分别放在哪里。
3. 根据任务类型进入对应目录，不要默认把所有报告都当成团队协作文档。

## 目录说明

- `团队协作/`：只放工作日志、接力说明、协作状态记录。适合了解“上一位智能体做到了哪里”。
- `数据审计/`：放训练数据、样本质量、覆盖率、特征集、盘口锚点和市场诊断报告。适合回答“数据是否可用、范围是什么、质量怎样”。
- `模型实验/`：放 baseline、模型对比、动态特征实验、HGB 对比、close-market baseline 等训练/评估结果。适合回答“模型表现怎样、下一步实验是什么”。
- `设计/`：放功能或模块设计文档。适合实现前理解结构和边界。
- `计划/`：放具体实施计划或阶段任务拆解。
- `规则/`：放项目协作规则。
- `备忘/`：放常用命令、回填规则等操作备忘。

## 写入约定

- 新的数据质量、覆盖率、QA、特征集报告写入 `docs/数据审计/`。
- 新的模型训练、模型对比、回测、指标评估报告写入 `docs/模型实验/`。
- 新的接力或工作日志写入 `docs/团队协作/`。
- 如果新增 CLI 默认报告路径，应同步更新 `Context.md` 和相关 Web/mock/test 路径。
