说明：本段接在 `jxlxy1995.home` 昨晚的 OddsPapi 回填进度、限流、别名与映射增强之后。目标是新增一个只读审计命令，把 worker 进度 JSON 和数据库里的回填状态聚合成一份终端可读报告，降低人工排查成本。

本轮完成内容：
- 新增 `src/icewine_prediction/oddspapi_backfill_audit_service.py`。
- 新增 CLI 命令 `odds-source oddspapi-audit-backfill`。
- 报告读取 `logs/odds/oddspapi-worker-progress.json` 展示当前 worker 状态、当前联赛、轮次、处理数、写入快照数、失败数和请求数。
- 报告按联赛聚合 `matches`、`odds_source_matches` 和 `historical_odds_snapshots`，展示已完赛比赛数、fixture 匹配数、有快照比赛数、快照总数、亚盘/大小球快照数、历史赔率状态计数和失败原因 Top N。
- 联赛名优先使用 `config/display_names.yaml` 中的中文名展示，同时保留英文原名，例如 `法乙 (Ligue 2)`。
- 新增设计文档 `docs/设计/20260527-OddsPapi回填审计汇总命令设计.md`。

使用方式：
```powershell
C:\Python312\python.exe src\icewine_cli.py odds-source oddspapi-audit-backfill `
  --season 2025 `
  --log-dir logs/odds `
  --top-errors 5
```

边界说明：
- 该命令只读，不调用 OddsPapi API。
- 不启动 worker。
- 不修改 `API_FOOTBALL_TO_ODDSPAPI_TOURNAMENT_IDS`。
- 不写入或更新 `odds_source_matches`。
- 不清理或重跑任何 historical odds。

给另一侧的话：
- worker 运行中可以用这个命令快速看当前跑到哪个联赛和哪一轮。
- worker 结束后可以用它看各联赛 `success`、`empty`、`unavailable`、`unmatched`、`failed` 的分布。
- 如果某个联赛失败原因集中，可以再决定是补球队别名、修 tournament 映射、缩小批次，还是调整 cooldown。
