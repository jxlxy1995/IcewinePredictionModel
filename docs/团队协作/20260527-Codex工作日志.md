使用方式：
```powershell
C:\Python312\python.exe src\icewine_cli.py odds-source oddspapi-audit-backfill `
  --season 2025 `
  --log-dir logs/odds `
  --top-errors 5
```

边界说明：
- 该命令只读，不调用 OddsPapi API。
- 不启动 worker。
- 不修改 `API_FOOTBALL_TO_ODDSPAPI_TOURNAMENT_IDS`。
- 不写入或更新 `odds_source_matches`。
- 不清理或重跑任何 historical odds。

给另一侧的话：
- worker 运行中可以用这个命令快速看当前跑到哪个联赛和哪一轮。
- worker 结束后可以用它看各联赛 `success`、`empty`、`unavailable`、`unmatched`、`failed` 的分布。
- 如果某个联赛失败原因集中，可以再决定是补球队别名、修 tournament 映射、缩小批次，还是调整 cooldown。

## 公司侧接力：OddsPapi 别名候选建议工具

作者：`xingyu.li`

日期：`2026-05-27`

当前基线提交：`48d7b73`

功能提交号：`54a94b2`

说明：本段继续接在 fixture 诊断失败分类之后，新增一个只读别名候选建议工具。它读取 `oddspapi-diagnose-fixtures` 产出的 `matches.jsonl`，从 `team_name_mismatch` 样本中提取可人工审核的 OddsPapi 球队别名候选。

本轮完成内容：
- 新增 `src/icewine_prediction/oddspapi_alias_suggestion_service.py`。
- 新增 CLI 命令 `odds-source oddspapi-suggest-aliases`。
- 输入为诊断 run 目录，例如 `logs/odds-diagnostics/<run_id>`。
- 输出可复制到 `config/external_aliases.yaml` 的 YAML 片段，以及每条建议的证据说明。
- 规则保持保守：只有一边队名达到 anchor 阈值，另一边低于 alias 阈值时，才建议低相似度那一边作为别名。
- 两边都低相似度时不输出建议，避免把整场错配误判成别名。
- 已存在于 `config/external_aliases.yaml` 的别名会跳过并计数。
- 新增设计文档 `docs/设计/20260527-OddsPapi别名候选建议工具设计.md`。

使用方式：
```powershell
C:\Python312\python.exe src\icewine_cli.py odds-source oddspapi-suggest-aliases `
  --report-dir logs/odds-diagnostics/<run_id> `
  --alias-config-path config/external_aliases.yaml `
  --alias-threshold 0.75 `
  --anchor-threshold 0.75
```

边界说明：
- 不调用 OddsPapi API。
- 不读取或写入数据库。
- 不自动修改 `config/external_aliases.yaml`。
- 不自动执行 `aliases add`。
- 不影响正式回填匹配策略。

给另一侧的话：
- 建议先按 `oddspapi-diagnose-fixtures` 跑出诊断报告，再用本命令生成候选别名。
- 输出的 YAML 片段需要人工审核后再复制到 `config/external_aliases.yaml`。
- 如果没有候选，通常说明没有可靠 anchor；此时应回到 fixture 诊断报告看 tournament 映射、时间窗口或附加赛归属。
