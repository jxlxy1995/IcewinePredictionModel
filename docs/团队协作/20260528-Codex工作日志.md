# 2026-05-28 Codex 工作日志

## 公司侧接力：胜平负历史赔率样本与 Web 曲线

作者：`xingyu.li`

日期：`2026-05-28`

当前基线提交：`53e5ea9`

功能提交号：`8b38759`

说明：本段接在隔壁 OddsPapi 回填器修复之后。隔壁已经把胜平负赔率以 `match_winner` 写入历史赔率快照，本轮公司侧把它接入历史赔率训练样本抽取入口，并让 Web 单场赔率曲线展示胜平负三项赔率。同时修复历史赔率曲线时间按 UTC 显示的问题，后端统一按北京时间输出快照时间。

本轮完成内容：
- `historical_training_sample_service.py` 从两市场扩展到三市场：
  - `asian_handicap`
  - `total_goals`
  - `match_winner`
- 胜平负样本要求同一时间点 `home`、`draw`、`away` 三项齐全。
- `HistoricalOddsAnchorFeature` 新增可选 `side_c`、`side_c_odds`、`side_c_implied_probability`、`side_c_result`。
- `HistoricalMarketTrainingSample` 新增可选 `side_c_odds_movement`。
- 胜平负结算只使用 `win` / `loss`，没有走水或半赢半输。
- `/api/matches/{match_id}/odds-trends` 新增 `match_winner` 曲线。
- Web 前端 `OddsTrendPanel` 新增胜平负走势展示。
- Web mock 和类型定义补齐 `match_winner`。
- 历史赔率曲线 `snapshot_time` 单独按 UTC -> `Asia/Shanghai` 输出，避免前端看到 UTC 时间。
- 新增设计文档 `docs/设计/20260528-胜平负历史赔率训练样本与Web曲线设计.md`。

边界说明：
- 不修改 OddsPapi 回填器。
- 不修改历史赔率数据库结构。
- 不重跑真实赔率数据。
- 不改变旧版 `training_sample_service.py`。
- 不把胜平负接入正式推荐策略。

给另一侧的话：
- 回填器重新跑完后，可以用 `samples historical-odds-preview --season 2026 --limit 20` 检查是否出现 `match_winner` 样本。
- 如果 `match_winner` 样本少，优先检查同一时间点是否缺 `home/draw/away` 任一项。
- Web 单场赔率走势现在应该显示北京时间；如果看到时间仍偏 8 小时，优先检查数据库里的 `snapshot_time` 是否已经被写成北京时间但没有时区。
