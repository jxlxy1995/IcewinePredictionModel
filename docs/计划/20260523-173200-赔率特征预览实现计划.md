# 赔率特征预览实现计划
创建时间：2026-05-23 17:32:00 北京时间

## 目标
新增第一版赔率特征生成与 CLI 预览能力，把已同步的 bookmaker 亚盘和大小球快照聚合为每场比赛可读、可供后续模型使用的市场特征。

## 范围
本阶段完成：

- 在 `feature_service.py` 中新增赔率聚合数据结构和服务函数。
- 聚合亚盘、主队赔率、客队赔率、大小球、大球赔率、小球赔率的均值、中位数、最小值、最大值和样本数。
- 计算盘口分歧程度，用最大盘口线减最小盘口线表达。
- 新增 `features preview --hours N` CLI，以中文比赛名展示未来窗口内已有赔率的比赛特征。

本阶段不完成：

- 正式推荐方向。
- 历史回测和训练。
- 完整盘口曲线表。
- Web 可视化。

## 文件

- 修改：`src/icewine_prediction/cli.py`

- 新增：`tests/test_features_cli.py`

## 实施步骤
1. 为赔率聚合写失败测试，覆盖均值、中位数、盘口分歧和空赔率处理。
2. 实现 `OddsMarketAggregate`、`MatchOddsFeatures` 和 `build_match_odds_features`。
3. 为未来窗口特征列表写失败测试，复用 `Match` 与 `OddsSnapshot`。
4. 实现 `list_upcoming_match_odds_features`。
5. 为 CLI 分组和输出写失败测试。
6. 新增 `features preview --hours N` 命令，展示中文比赛名、亚盘共识、大小球共识和样本数。
7. 运行全量测试，做一次本地 CLI 预览冒烟。
8. 使用中文提交记录提交。

## 历史学习衔接
后续开通会员拿到历史赛程、赛果和历史赔率后，本阶段字段会成为训练样本的基础输入。每一条历史比赛可以按开赛前某个时间点生成相同的赔率特征，再与赛果、亚盘结算、大小球结算拼成监督学习样本。
