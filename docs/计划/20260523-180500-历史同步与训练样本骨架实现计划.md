# 历史同步与训练样本骨架实现计划
创建时间：2026-05-23 18:05:00 北京时间

## 目标
新增历史赛程同步和第一版训练样本生成骨架，为后续开通付费套餐后批量回填历史比赛、训练模型做准备。

## 范围
本阶段完成：

- 支持按 API-Football 联赛 ID 和赛季拉取历史 fixtures。
- 复用现有 `upsert_fixtures` 写入 `matches`、`leagues`、`teams`。
- 新增 CLI：`sync history --league-id 135 --season 2024`。
- 新增训练样本生成服务，从已完赛比赛生成 dataclass 样本。
- 新增 CLI：`samples preview --limit 10`，展示已有样本标签。

本阶段不完成：

- 批量多联赛多赛季调度。
- 历史赔率回填。
- 数据库训练样本表。
- 机器学习训练命令。
- 赛前滚动特征和球队近期状态特征。

## 设计
历史同步：

- provider 新增 `fetch_historical_fixtures(league_id, season)`。
- API 请求使用 `fixtures?league=<id>&season=<season>&timezone=Asia/Shanghai`。
- 免费权限可能返回 season 限制错误，当前只让错误暴露为可读异常，不伪造数据。

训练样本：

- 第一版样本只使用已经存在的完赛比赛。
- 样本字段包含比赛 ID、联赛、主客队、开赛时间、比分、赛果标签、总进球。
- 如果比赛有赔率快照且盘口线存在，额外生成亚盘主队/客队结算标签和大小球大/小结算标签。
- 样本不落库，先用于预览和后续训练管线接口稳定。

## 验证

- provider 单测确认请求参数。
- sync runner 单测确认汇总格式。
- sample service 单测确认完赛过滤和结算标签。
- CLI 单测确认命令分组存在和格式化输出。
- 全量测试通过后提交中文 commit。
