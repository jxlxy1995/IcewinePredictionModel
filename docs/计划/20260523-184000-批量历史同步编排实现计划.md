# 批量历史同步编排实现计划
创建时间：2026-05-23 18:40:00 北京时间

## 目标
新增保守的批量历史同步编排命令，按联赛配置和赛季范围由近到远回填历史赛果，并在批次后可选补近期历史赔率。

## 命令

```powershell
python -m icewine_cli sync backfill-history --from-season 2024 --to-season 2024 --max-leagues 1 --historical-odds-days 3
```

## 规则
- 只使用 `config/leagues.yaml` 中 `enabled = true` 的联赛。
- 联赛按 `priority` 从高到低排序。
- `--max-leagues` 默认 1，避免开发阶段打穿 API 请求额度。
- 赛季按从近到远排序，例如 `2024, 2023, 2022`。
- 每个联赛每个赛季调用一次历史 fixtures 同步。
- 如果 `--historical-odds-days > 0`，批次结束后调用一次近期历史赔率同步。
- 输出逐项摘要，方便人工判断请求数和新增量。

## 边界
本阶段不做：

- 自动恢复断点。
- 后台任务调度。
- 复杂失败重试。
- 历史赔率按赛季补齐。

## 验证
- 单测覆盖联赛排序、赛季倒序、最大联赛数限制。
- 单测覆盖 runner 输出多行摘要。
- CLI 单测覆盖命令选项。
- 真实只跑小范围冒烟，避免额外消耗过多请求。
