# OddsPapi 回填批次状态说明

## 背景

`config/oddspapi_backfill_batches.yaml` 只保留当前明确准备自动回填的批次。已经跑过的联赛、只需要补漏的联赛、以及尚未通过小样本验证的联赛，都不再放入自动批次，避免误启动 worker 后重复扫库、重复试探或消耗请求。

## 当前自动回填批次

当前为空。

```yaml
batches: []
```

只有当某个联赛已经确认需要继续自动回填，并且小样本验证能稳定写入赔率快照时，才重新加入配置。

## 后续排查顺序

1. 先生成联赛级覆盖率报表，确认哪些联赛只是“已完成但覆盖不足”。
2. 对待复测联赛逐个做 1-5 场小样本验证。
3. 若未匹配，优先检查：
   - OddsPapi tournament id 是否正确。
   - API-Football 队名与 OddsPapi 队名是否需要别名。
   - 开赛时间窗口是否偏窄。
4. 若返回 404 或卡在盘口定义请求，优先检查：
   - OddsPapi fixture id 是否可用。
   - 该 fixture 是否支持 Pinnacle 足球盘口。
   - 请求是否需要更短的单轮超时和失败熔断。
5. 单联赛小样本确认可以写入快照后，再把该联赛加入 `config/oddspapi_backfill_batches.yaml`。

## 启动原则

不要直接重跑历史批次。后续启动 worker 前先确认：

- `batches` 中只包含本轮真的要跑的联赛。
- 每个联赛已经通过小样本验证。
- 命令带有单轮超时和连续失败熔断参数。

建议默认参数：

```powershell
--round-timeout-seconds 60 --stop-after-failed-rounds 2
```
