# Web 控制台中文名交互与 mock 增强实现计划

## 背景

`jxlxy1995.home` 在团队协作日志中建议公司侧优先推进不依赖真实 `.env`、SQLite 和 API key 的 Web 控制台工作。本轮接手中文名维护页面交互与 mock fallback，避免触碰 OddsPapi 回填器和真实数据回填线。

## 范围

- 增强中文名维护页筛选能力：全部球队、只看未翻译、只看已修改。
- 保存按钮增加草稿判断、保存中禁用态和保存中展示。
- 切换联赛前若存在未保存草稿，提示用户确认。
- 扩充 mock 数据，让无后端环境也能演示中文名页面的已翻译、未翻译、已完成联赛、球队 logo 和保存后展示。
- 增加前端轻量测试，覆盖中文名工作台纯逻辑。

## 不做

- 不调用真实 API-Football、OddsPapi 或 The Odds API。
- 不修改 OddsPapi worker、批量回填、历史赔率抽样规则。
- 不提交本地 SQLite、日志、`web/node_modules` 或 `web/dist`。

## 验证

- `cd web && npm run test -- displayNameWorkspace`
- `cd web && npm run build`
