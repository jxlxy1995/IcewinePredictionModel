# 历史赔率样本质量报告实现计划

日期：2026-05-27

## 目标

新增历史赔率训练样本质量报告，默认用 `2026-01-15 00:00 Asia/Shanghai` 之后的已完赛有比分比赛作为训练可用覆盖率分母，同时保留整赛季覆盖率对照。

## 实现步骤

1. 新增报告服务测试。
   - 构造同一联赛 3 场比赛：1 场早于 eligible start，2 场晚于 eligible start。
   - 只给其中 1 场 eligible 比赛写入 historical odds snapshot。
   - 断言 eligible 覆盖率为 `0.5000`，full-season 覆盖率为 `0.3333`。

2. 新增 `historical_training_sample_report_service.py`。
   - 定义 report dataclass。
   - 查询已完赛、有比分比赛。
   - 复用 `list_historical_market_training_samples` 生成历史赔率市场样本。
   - 统计总览、按市场、按联赛三层指标。

3. 接入 CLI。
   - 新增 `samples historical-odds-report`。
   - 支持 `--season`、`--bookmaker`、`--eligible-start`。
   - `--eligible-start` 按北京时间解析。

4. 补充 CLI 测试。
   - `samples --help` 能看到新命令。
   - 命令能把 eligible start 传入报告服务，并输出覆盖率摘要。

5. 补设计文档和工作日志。
   - 说明覆盖率分母口径。
   - 说明本功能只读，不改回填器和抽样规则。

6. 验证。
   - 运行相关 pytest。
   - 运行全量 pytest。
   - 运行 ruff check。
