# 历史表现接入推荐解释设计

## 目标

在推荐预览中展示“类似历史样本”的表现，让出手前能看到同类推荐过去的样本数和 ROI。第一版只作为解释信息，不改变模型概率、信心等级、是否出手和手数。

## 关键边界

`Recommendation.similar_backtest_roi` 当前已经参与信心等级计算，并且模型推荐里暂时传入固定值 `0.05`。本阶段不把真实历史 ROI 写入这个字段，避免悄悄改变推荐结果。新增独立展示字段：

- `historical_sample_count`
- `historical_roi`

这两个字段只用于 CLI 展示，不参与推荐计算和记录落库。

## 相似条件

V1 的相似样本过滤条件采用：

- 盘口类型：`market_type`
- 推荐方向：`side`
- edge 分桶：由当前推荐 `edge` 映射

暂不强制加入联赛和信心等级。原因是早期样本量少，如果条件过细，大多数比赛会显示 0 样本。后续样本变多后，可以把联赛、信心等级做成更细的解释层级。

## 架构

新增一个推荐解释增强服务，负责：

1. 接收当前推荐列表。
2. 对每条非观望推荐构造 `HistoricalPerformanceFilters`。
3. 调用 `build_historical_performance_report`。
4. 用 `dataclasses.replace` 返回带展示字段的新推荐对象。

CLI 的 `recommendations model-preview` 和 `recommendations record` 都可以使用这一步。`record` 虽然落库时不保存展示字段，但这样后续若记录前需要展示或调试，路径保持一致。

## 展示

推荐行中新增片段：

```text
历史 12场 ROI 0.0833
```

如果没有历史样本，则展示：

```text
历史 0场
```

## 自检

- 不改变出手手数。
- 不改变信心等级。
- 不改变推荐记录表结构。
- 不读取待结算推荐作为历史样本。
- 展示文本使用中文；代码命名保持英文。
