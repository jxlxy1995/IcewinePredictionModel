# 相似历史表现服务设计

## 目标

在推荐记录已经结算后，按相似条件聚合历史表现，帮助判断某类推荐过去是否真正产生正收益。第一版只提供观察能力，不直接调整模型概率、信心等级或出手手数。

## 范围

本阶段统计数据来源为 `RecommendationRecord`，只纳入 `status = settled` 的记录。支持可选过滤条件：

- 盘口类型：`asian_handicap`、`total_goals`
- 推荐方向：`home`、`away`、`over`、`under`
- 联赛名称
- edge 分桶：`0.00-0.03`、`0.03-0.06`、`0.06-0.10`、`0.10+`
- 信心等级：如 `B+`、`A-`

## 输出

服务输出一个历史表现摘要：

- 样本数
- 总手数
- 总盈亏
- ROI
- 按结算结果分布：赢、半赢、走水、半输、输
- 按 edge 分桶表现
- 按盘口类型表现
- 按方向表现
- 按信心等级表现
- 按联赛表现

## 架构

新增 `historical_performance_service.py`，负责查询、过滤和聚合。CLI 只负责接收参数并格式化展示，不直接写聚合逻辑。edge 分桶复用 `record_service.edge_bucket_for_value`，汇总结构复用 `RecordGroupSummary`，避免重复定义 ROI 计算规则。

## CLI

新增命令：

```powershell
python -m icewine_cli records performance
```

可选参数：

```powershell
--market-type asian_handicap
--side home
--league-name "La Liga"
--edge-bucket 0.06-0.10
--confidence-grade B+
```

输出文本保持中文；模型和数据库字段继续使用英文。

## 后续衔接

当历史样本达到一定规模后，可以将该服务输出接入推荐解释，展示“类似历史样本 ROI”。再往后才考虑把历史表现作为手数校准因子，避免样本不足时过拟合。

## 自检

- 无数据库结构变更。
- 不读取未结算推荐，避免未来比赛污染统计。
- 不改变当前推荐生成逻辑。
- 时间相关逻辑未新增，仍遵守北京时间规则。
