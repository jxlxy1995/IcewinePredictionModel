# 联赛白名单与覆盖率报告设计

## 目标

建立主流联赛 API 白名单和中文展示名，并新增本地历史数据覆盖率报告，为 25/26 赛季批量拉取、回测和训练做准备。

## 配置位置

- API 联赛白名单：`config/leagues.yaml`
- 中文展示名：`config/display_names.yaml`

模型层和同步层继续使用 API 返回的英文联赛名与 league id。CLI 展示层通过 `DisplayNameService` 显示中文译名。附加赛、升级赛、降级赛等特殊赛事的归属完全按 API 返回的 league id/name 处理，不额外合并。

## 白名单范围

第一批覆盖：

- 欧洲五大联赛和主要二级联赛。
- 葡超、荷甲、比甲、苏超、土超、奥甲、瑞士超等欧洲主流联赛。
- 欧冠、欧联、欧协联。
- 日职联、韩K联、澳超、沙特联、中超、美职联等非欧洲主流联赛。

每个条目包含：

- `name`: API 官方英文名
- `country`: API 国家/地区
- `api_football_id`: API league id
- `enabled`: 是否参与默认同步
- `priority`: 批量任务优先级

## 覆盖率报告

新增 `history coverage` CLI 命令，第一版只统计本地数据库，不在报告命令里请求外部 API。

每个联赛输出：

- 总比赛数
- 已完赛数
- 有比分数
- 有赔率比赛数
- 有亚盘比赛数
- 有大小球比赛数
- 赔率覆盖率
- 亚盘覆盖率
- 大小球覆盖率

报告可选按赛季过滤。赛季按开球时间年份近似统计，后续如果需要更精确，可在数据库中新增 season 字段。

## 自检

- 不在覆盖率报告里消耗 API 请求。
- 不改变已有同步逻辑。
- 中文译名放展示配置，不污染模型字段。
- 特殊赛事归属按 API 返回处理。
