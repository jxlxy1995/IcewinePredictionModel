# 负二项总进球模型设计

创建时间：2026-05-25 北京时间

## 一、目标

本阶段新增负二项总进球模型，用于从历史比赛总进球估计更宽的总进球分布，并计算大小球覆盖概率。

普通泊松分布默认方差等于均值。足球比赛的总进球在不同联赛、不同赛程和不同比赛状态下可能出现过度离散，即方差大于均值。负二项模型用于表达这种更宽的尾部分布。

## 二、输入与边界

输入只使用 `TrainingSample` 中已经存在的字段：

- `total_goals`
- `time_decay_weight`

本阶段不读取赔率快照，不使用 24 小时赔率走势，不改默认推荐逻辑，不做球队级负二项攻防模型。

## 三、模型形式

模型保存：

- 总进球均值 `mean_goals`
- 离散度 `dispersion`

采用常见参数化：

```text
variance = mean + dispersion * mean^2
```

当样本方差接近或小于均值时，使用很小的离散度下限，使模型接近泊松分布。

## 四、大小球概率

模型支持：

- 整数线
- 半球线
- 四分之一线

结算口径：

- 赢为 `1`
- 走水为 `0.5`
- 输为 `0`
- 四分之一线拆成两个相邻半盘口后取平均

输出：

- 大球覆盖概率
- 小球覆盖概率

## 五、模块设计

新增 `negative_binomial_model_service.py`。

核心类型：

```python
NegativeBinomialTotalGoalsModel
NegativeBinomialTotalGoalsProbability
```

核心函数：

```python
train_negative_binomial_total_goals_model(samples)
```

底层使用 `scipy.stats.nbinom`。

## 六、CLI 设计

新增训练命令：

```powershell
icewine models train-negative-binomial-total --limit 1000
```

新增纯参数调试命令：

```powershell
icewine models negative-binomial-total --mean 2.60 --dispersion 0.20 --line 2.75
```

训练命令读取数据库训练样本；调试命令不读取数据库。

## 七、明确不做

本阶段不做：

- 不做主队/客队双负二项模型。
- 不做球队攻防强度负二项模型。
- 不接入默认推荐。
- 不接入赔率走势。
- 不声明收益提升。

后续可以在数据积累和回测稳定后，考虑联赛级负二项或与 Dixon-Coles/Skellam 的模型对比报告。
