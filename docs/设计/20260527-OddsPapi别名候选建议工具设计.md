# OddsPapi 别名候选建议工具设计

## 背景

fixture 诊断报告已经能把失败分类到 `team_name_mismatch`。这类样本通常需要人工补 OddsPapi 球队别名，但直接从候选 fixture 里手抄队名容易漏，也容易把整场错配误当成别名。

本设计新增一个只读建议工具，从 `oddspapi-diagnose-fixtures` 产出的 `matches.jsonl` 中提取候选别名片段，供人工审核后复制到 `config/external_aliases.yaml`。

## 命令

```powershell
C:\Python312\python.exe src\icewine_cli.py odds-source oddspapi-suggest-aliases `
  --report-dir logs/odds-diagnostics/<run_id> `
  --alias-config-path config/external_aliases.yaml `
  --alias-threshold 0.75 `
  --anchor-threshold 0.75
```

## 输入

工具读取诊断目录下的：

```text
matches.jsonl
```

只处理 `failure_category=team_name_mismatch` 的比赛。

## 建议规则

为了避免把整场错配误判成别名，规则保持保守：

- 如果主队相似度低于 `alias_threshold`，但客队相似度达到 `anchor_threshold`，则建议主队别名。
- 如果客队相似度低于 `alias_threshold`，但主队相似度达到 `anchor_threshold`，则建议客队别名。
- 如果主客队都低于 anchor，不输出建议，认为更可能是整场候选 fixture 错配。
- 如果建议别名已经存在于 `config/external_aliases.yaml`，跳过并计数。
- 如果规范化后的 canonical name 和 alias name 相同，跳过。

## 输出

输出包含三段：

1. 汇总信息
   - 诊断报告目录。
   - 当前别名配置路径。
   - 建议数量。
   - 已存在别名跳过数量。

2. 可复制 YAML 片段

```yaml
aliases:
  - entity_type: team
    source_name: oddspapi
    canonical_name: Wolves
    alias_name: Wolverhampton Wanderers
```

3. 证据说明
   - `match_id`
   - 联赛名
   - OddsPapi fixture id
   - 建议方主客队
   - anchor 方主客队
   - 两侧相似度

## 边界

- 不调用 OddsPapi API。
- 不读取或写入数据库。
- 不自动修改 `config/external_aliases.yaml`。
- 不自动执行 `aliases add`。
- 不影响正式回填匹配策略。

## 使用建议

先运行 fixture 诊断，再对对应 run 目录执行本工具。人工审核 YAML 片段时，优先关注证据中 anchor 相似度高、fixture id 与比赛确实对应的建议。两边都低相似度的错配不会被输出，需要回到 fixture 诊断报告查看 tournament 映射、时间窗口或附加赛归属。
