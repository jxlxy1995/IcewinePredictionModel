# OddsPapi 回填审计汇总命令设计

## 背景

OddsPapi 回填过程中，人工排查需要同时查看 worker 进度 JSON、日志尾部、`odds_source_matches` 状态和 `historical_odds_snapshots` 覆盖情况。信息分散时，很难快速判断当前是“正常跑着”“需要补别名”“fixture 不可用”还是“历史赔率接口异常”。

本设计新增一个只读审计命令，用于把这些信息汇总成一份终端可读报告。它不请求 OddsPapi，不启动 worker，不修改数据库。

## 命令

```powershell
C:\Python312\python.exe src\icewine_cli.py odds-source oddspapi-audit-backfill `
  --season 2025 `
  --log-dir logs/odds `
  --top-errors 5
```

## 数据来源

- `logs/odds/oddspapi-worker-progress.json`
  - 展示 worker 当前状态、当前联赛、轮次、已处理比赛、写入快照、失败数和请求数。
- `matches`
  - 按赛季筛选已完赛比赛，作为联赛汇总分母。
- `odds_source_matches`
  - 统计每个联赛的 fixture 匹配数和历史赔率状态分布。
- `historical_odds_snapshots`
  - 统计每个联赛已写入的历史赔率快照数、亚盘快照数、大小球快照数和有快照的比赛数。
- `config/display_names.yaml`
  - 联赛名优先使用中文展示，保留英文原名作为括号说明，例如 `法乙 (Ligue 2)`。

## 输出内容

报告分两段：

1. Worker 进度
   - 当前 worker 状态、更新时间、模式、赛季。
   - 当前联赛使用中文名展示。
   - 当前轮次和累计处理指标。

2. 联赛汇总
   - 每个联赛的已完赛比赛数。
   - 已匹配 fixture 的比赛数。
   - 有历史赔率快照的比赛数。
   - 历史赔率快照总数、亚盘快照数、大小球快照数。
   - `success`、`empty`、`unavailable`、`unmatched`、`failed` 等状态计数。
   - 每个联赛最近常见失败原因 Top N。

## 边界

- 该命令只读，不会调用 OddsPapi API。
- 不会修改 `API_FOOTBALL_TO_ODDSPAPI_TOURNAMENT_IDS`。
- 不会写入或更新 `odds_source_matches`。
- 不会清理或重跑任何 historical odds。
- 如果没有 worker 进度快照，仍会输出数据库侧联赛汇总。

## 使用建议

回填 worker 运行中，可以先看该命令判断当前是否继续推进；worker 停止后，也可以用它快速定位下一步人工处理方向：

- `unmatched` 多：优先看 fixture 诊断和球队别名。
- `unavailable` 多：优先看 fixture 是否真实支持历史赔率，或是否混入附加赛/特殊分组。
- `failed` 多且错误含 `429`：优先降低并发、增大 cooldown 或缩小批次。
- 快照数增长但 `empty` 多：优先确认盘口定义和 Pinnacle 覆盖情况。
