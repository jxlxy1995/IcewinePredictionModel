# 历史赔率样本质量报告设计

日期：2026-05-27

## 背景

OddsPapi 当前可稳定回填的足球历史赔率主要从 2026-01-15 之后开始。训练样本质量报告如果继续用整赛季比赛数作为唯一分母，会把数据源历史不可用的问题和真实回填缺口混在一起，导致覆盖率偏低且难以排错。

本设计新增一个只读报告层，用于回答两个问题：

- 从 2026-01-15 之后看，已完赛有比分比赛中有多少场已经能生成历史赔率训练样本。
- 如果按整赛季看，2026-01-15 之前的数据缺口会对覆盖率造成多大影响。

## 口径

默认 eligible start 为 `2026-01-15 00:00 Asia/Shanghai`。

分母分为两层：

- `eligible_match_count`：开赛时间不早于 eligible start 的已完赛、有比分比赛。
- `full_season_match_count`：指定赛季内全部已完赛、有比分比赛。

覆盖率分为两层：

- `eligible_coverage_ratio = 有历史赔率样本的 eligible 比赛数 / eligible_match_count`。
- `full_season_coverage_ratio = 有历史赔率样本的 eligible 比赛数 / full_season_match_count`。

`excluded_before_eligible_start_count` 单独展示，说明有多少比赛因为早于数据源可用起点而没有纳入训练可用覆盖率分母。

## 服务边界

新增 `historical_training_sample_report_service.py`，只依赖：

- `Match`：统计已完赛、有比分比赛分母。
- `list_historical_market_training_samples`：复用已经定义好的历史赔率抽样规则。

报告层不修改：

- OddsPapi 回填器。
- 历史赔率快照。
- 历史赔率抽样规则。
- 旧版 `training_sample_service.py`。

## 输出内容

总览：

- 赛季。
- bookmaker。
- eligible start。
- full-season match count。
- eligible match count。
- excluded-before-start count。
- match-with-sample count。
- eligible coverage。
- full-season coverage。

按盘口市场：

- 样本数。
- 完整锚点样本数。
- `thin_history` 样本数。
- 平均快照数。
- 平均绝对盘口变化。
- 缺失锚点分布。

按联赛：

- full-season 比赛数。
- eligible 比赛数。
- excluded-before-start 比赛数。
- 有样本比赛数。
- eligible 覆盖率。
- full-season 覆盖率。

## CLI

新增命令：

```powershell
C:\Python312\python.exe src\icewine_cli.py samples historical-odds-report `
  --season 2026 `
  --bookmaker pinnacle `
  --eligible-start 2026-01-15
```

`--eligible-start` 接受 ISO date 或 ISO datetime；没有时区时按北京时间解释。

## 后续用途

这份报告可以作为建模前检查：

- 如果 eligible 覆盖率低，优先继续排查回填、fixture 匹配或 bookmaker 数据缺口。
- 如果 eligible 覆盖率高但 `thin_history` 很多，说明比赛可生成样本，但 24h 内快照密度不足。
- 如果缺失集中在 `24h`，偏向早段赔率不足；如果缺失集中在 `close`，需要检查 close cutoff 或赛前过滤。
