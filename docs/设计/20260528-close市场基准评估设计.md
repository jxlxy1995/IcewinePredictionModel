# close 市场基准评估设计

日期：`2026-05-28`

## 背景

历史赔率回填已经开始覆盖三类市场：

- `asian_handicap`
- `total_goals`
- `match_winner`

在正式训练“盘口运动模型”或“close 市场融合模型”之前，需要先回答一个更基础的问题：如果只使用临场 close 赔率本身，市场对赛果/盘口结算的解释力大概是什么水平。

本设计新增只读评估层，不改回填器、不写数据库、不改变推荐策略。它把 close implied probability 当作一个市场基准模型，用于后续判断自研模型是否真的超过市场基准。

## 评估口径

输入来自 `historical_odds_feature_service.py` 的 `HistoricalOddsMarketFeature`。

对每条特征：

1. 读取 close anchor 的 implied probability。
2. 按同一市场内概率和归一化，去除 overround 影响。
3. 根据 close anchor 的结算结果确定真实类别。
4. 计算：
   - `log_loss`
   - `brier`
   - `accuracy`
   - `overround`

不同市场的类别定义：

- `match_winner`：三分类，`home/draw/away`。
- `asian_handicap`：二分类，`home/away`。
- `total_goals`：二分类，`over/under`。

`match_winner` 只评估“恰好一个 side 为 `win`，其余 side 为 `loss`”的样本。

`asian_handicap` 和 `total_goals` 使用盘口结算强度评估：

- `win`：side A 目标值 `1.00`
- `half_win`：side A 目标值 `0.75`
- `half_loss`：side A 目标值 `0.25`
- `loss`：side A 目标值 `0.00`

二分类市场的 `log_loss` 使用 soft-label binary cross entropy，`brier` 使用 soft-label Brier。`accuracy` 只判断方向：`win/half_win` 算 side A 方向，`loss/half_loss` 算 side B 方向。

`push`、缺概率或缺结算的样本计入 skipped，不进入指标均值。这个约束保留走水的中性语义，避免把无输赢样本硬压成方向胜负。

## 服务层

新增 `close_market_baseline_service.py`。

核心入口：

- `build_close_market_baseline_report(features)`
- `build_close_market_baseline_report_from_session(session, season, limit, bookmaker)`
- `format_close_market_baseline_report(report)`

报告结构：

- 总特征数
- 已评估样本数
- 跳过样本数
- 每个市场的 feature/evaluated/skipped
- 每个市场的平均 `log_loss`
- 每个市场的平均 `brier`
- 每个市场的 `accuracy`
- 每个市场的平均 `overround`

即使某个市场当前没有样本，也会保留在报告中，避免回填缺口被误解为“无需检查”。

## CLI

新增命令：

```powershell
C:\Python312\python.exe src\icewine_cli.py samples historical-odds-close-baseline `
  --season 2026 `
  --bookmaker pinnacle
```

可选参数：

- `--season`
- `--limit`
- `--bookmaker`

建议使用方式：

1. 回填完成后先跑 `samples historical-odds-report` 看覆盖率。
2. 再跑 `samples historical-odds-features-preview` 抽查概率、overround 和 close 结算。
3. 最后跑 `samples historical-odds-close-baseline` 看 close 市场基准。

## 边界

- 不修改 OddsPapi 回填器。
- 不修改数据库表结构。
- 不重新拉取历史赔率。
- 不训练新模型。
- 不把输出接入正式推荐策略。
- 走水 `push` 暂不进入指标均值，后续如需要可单独增加中性样本统计。
