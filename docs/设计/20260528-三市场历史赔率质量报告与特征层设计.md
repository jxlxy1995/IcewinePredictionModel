# 三市场历史赔率质量报告与特征层设计

日期：2026-05-28

## 背景

OddsPapi 回填器已经支持三类历史赔率市场：

- `asian_handicap`
- `total_goals`
- `match_winner`

在隔壁重新回填数据期间，模型侧不适合马上训练融合模型。更稳妥的下一步是先增强只读质量报告，并提供一个统一的历史赔率特征层，方便回填完成后快速判断数据能不能进入模型训练。

## 质量报告增强

`samples historical-odds-report` 保留原来的 eligible 分母口径：

- 默认 eligible start：`2026-01-15 00:00 Asia/Shanghai`
- 分母：eligible start 之后已完赛、有比分比赛

本轮增强每个市场的独立覆盖指标。即使某个市场没有样本，也会出现在报告中，避免 `match_winner` 缺失时被误以为没有检查。

新增市场级指标：

- `eligible_match_count`
- `sample_count`
- `sample_coverage_ratio`
- `complete_anchor_sample_count`
- `complete_anchor_coverage_ratio`
- `close_anchor_sample_count`
- `close_anchor_coverage_ratio`
- `quality_tag_counts`

用途：

- `sample_coverage_ratio` 用于判断该市场是否已具备训练入口。
- `complete_anchor_coverage_ratio` 用于判断完整 24h 锚点是否足够。
- `close_anchor_coverage_ratio` 用于判断 close 市场融合能否先做。
- `quality_tag_counts` 用于定位 `thin_history` 等数据质量问题。

## 特征层

新增 `historical_odds_feature_service.py`。

它从 `HistoricalMarketTrainingSample` 生成只读特征，不访问 OddsPapi，不写数据库，不训练模型。

核心输出 `HistoricalOddsMarketFeature` 包含：

- 比赛、联赛、球队、比分信息。
- 市场类型和 bookmaker。
- opening anchor 与 close anchor 标签。
- opening line、close line、line movement。
- close implied probability。
- implied probability movement。
- opening overround 与 close overround。
- odds movement。
- close anchor 的结算结果。
- 快照数、缺失锚点和质量标签。

胜平负样本使用：

- `side_a = home`
- `side_b = draw`
- `side_c = away`

亚盘和大小球仍只使用 `side_a` / `side_b`。

## CLI

新增只读预览命令：

```powershell
C:\Python312\python.exe src\icewine_cli.py samples historical-odds-features-preview `
  --season 2026 `
  --limit 20 `
  --bookmaker pinnacle
```

这个命令适合在回填完成后快速确认：

- 是否有 `match_winner` 特征。
- close implied probability 是否合理。
- probability movement 是否存在异常大跳动。
- `thin_history` 是否集中出现。

## 边界

- 不修改 OddsPapi 回填器。
- 不修改数据库结构。
- 不重跑真实赔率数据。
- 不训练模型。
- 不改变正式推荐策略。
- 不替代后续真正的模型训练数据集，只作为特征层雏形和数据验收入口。
