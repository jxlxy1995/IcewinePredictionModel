**总公式**
每个市场先算两个方向，比如亚盘是 `home_cover / away_cover`，大小球是 `over / under`。

市场概率：

```text
raw_market_p_i = 1 / odds_i
market_p_i = raw_market_p_i / sum(raw_market_p_all_sides)
```

也就是先把赔率转成隐含概率，再归一化去掉水位影响。

模型概率：

```text
model_p_i = HGB模型.predict_proba(features)[i]
```

edge：

```text
edge_i = model_p_i - market_p_i
```

系统会选 edge 最大的方向：

```text
selected_side = argmax(model_p_i - market_p_i)
```

然后看这个方向是否满足对应策略阈值。

**模型本身**
当前纸面队列用的是：

```text
raw_hgb_team_form_plus_all_markets
```

也就是：

```text
SimpleImputer(strategy="median")
+ HistGradientBoostingClassifier(
    l2_regularization=0.05,
    max_iter=100,
    random_state=42
  )
```

特征是 `TEAM_FORM_FEATURES + ALL_MARKET_FEATURES`，大体包括：

- 主客队赛前历史战绩：场数、场均积分、胜平负率、进失球、主/客场积分、休息天数
- 胜平负赔率隐含概率
- 亚盘盘口、主客方向隐含概率、overround
- 大小球盘口、大小方向隐含概率、overround

注意：目前 **没有单独的胜平负推荐信号**。胜平负赔率只是模型输入特征之一。

**信号 1：亚盘客队方向 · HGB边际 v1**
策略 key：

```text
asian_away_cover_hgb_edge_v1
```

公式：

```text
先用亚盘模型得到:
edge_home_cover = model_p_home_cover - market_p_home_cover
edge_away_cover = model_p_away_cover - market_p_away_cover

选择 edge 更大的方向。
只有 selected_side == away_cover 且 edge >= 0.1000 才是 candidate。
```

换句话说，它只要“客队赢盘”方向，不要主队方向。

推荐盘口展示公式：

```text
如果亚盘 close_line = L
客队推荐盘口 = -L
```

例如 `line = -0.50`，展示为 `客队 +0.50`。

**信号 2：亚盘客队方向 · HGB分盘口桶 v2**
策略 key：

```text
asian_away_cover_hgb_bucket_v2
```

它和信号 1 使用同一个亚盘模型、同一个 edge 公式，但阈值按盘口桶不同：

```text
away_favorite: edge >= 0.1500
away_underdog: edge >= 0.2000
pickem: edge >= 0.0800
```

盘口桶代码含义：

```text
line > 0   => away_favorite   客队让球/客队更强侧
line == 0  => pickem          平手盘
line < 0   => away_underdog   客队受让/客队弱势侧
```

所以 v2 不是新模型，而是“同一个亚盘客队 edge，按盘口类型使用不同阈值”。

**信号 3：大小球方向 · HGB分盘口桶 v2**
策略 key：

```text
total_goals_hgb_bucket_v2
```

公式：

```text
edge_over = model_p_over - market_p_over
edge_under = model_p_under - market_p_under

selected_side = over / under 中 edge 更大的方向
```

但当前只开放 `2.75` 这个盘口桶：

```text
over@mid_2.75: edge >= 0.0800
under@mid_2.75: edge >= 0.0800
```

盘口桶代码：

```text
line <= 2.25  => low_<=2.25
line == 2.50  => mid_2.50
line == 2.75  => mid_2.75
line >= 3.00  => high_>=3.00
```

所以目前大小球真正会进入候选的只有：

```text
大小球盘口 = 2.75
且 over 或 under 的 edge >= 0.0800
```

**术语小抄**

- **HGB**：HistGradientBoostingClassifier，直方图梯度提升树，类似一组决策树逐步修正预测误差。
- **edge**：模型概率减去市场概率。比如模型认为客队赢盘 65%，市场概率 50%，edge = 15%。
- **market probability / 市场概率**：从赔率反推出来的概率，并做归一化。
- **overround / 水位超额**：庄家赔率隐含概率总和超过 100% 的部分。
- **away_cover / 客队赢盘**：客队按亚盘结算后赢。
- **pickem / 平手盘**：盘口为 0。
- **line bucket / 盘口桶**：把盘口按类型分组，用不同阈值筛选。
